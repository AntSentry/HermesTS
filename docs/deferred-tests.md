# Deferred Tests

Upstream test files that exercise core functionality but cross-import modules from packages that haven't been ported yet. Each downstream porter (state, agent, tools, cli, etc.) MUST pull from this list when claiming their task.

When you port a downstream package and unblock a deferred test, port it as part of that workstream, delete its row from this table, and append a `// Ported from <upstream path>` comment to the test file. Once the table is empty the migration is fully covered.
